Topic: RETRO

Recall

relative position encoding and absolute position encoding
Why does the chunked cross attention work on top of overlapped chunks to compute the attention vectors?

Notes

In the original self-attention mechanism, the attention score between two positions i and j is computed as:
Attention(Q, K) = softmax(Q * K^T / sqrt(d_k))
where Q, K are the query and key vectors, and d_k is their dimensionality.
In the modified self-attention mechanism that uses relative position representations, the attention score between two positions i and j is computed as:
Attention(Q, K) = softmax((Q * K^T + a(i-j)) / sqrt(d_k))
where a(i-j) is a learned embedding for the relative position (i-j).
So, the main difference is that in absolute position encodings, the position information is added to the input embeddings before the self-attention mechanism. In contrast, in relative position encodings, the position information is integrated directly into the self-attention mechanism itself.
...
The overlapping of chunks when computing attention vectors in a transformer-based model like the RETRO model is a strategy to deal with the limitation of the transformer's fixed-length context window and to maintain the continuity of the context.
In transformer models, the maximum sequence length that a model can handle is typically fixed due to memory constraints during training. This means that when processing long sequences of data, these sequences need to be divided into smaller "chunks" that fit within the model's maximum sequence length.
If chunks were processed independently, without overlap, then the model would not have any context from the previous chunk when processing the current chunk. This could lead to discontinuities in the model's understanding of the sequence, as it wouldn't be able to relate the context of the current chunk to the context of the previous chunk.
By introducing overlap between the chunks, the model is able to "carry over" some context from the previous chunk into the current chunk. This helps to maintain the continuity of the context across the boundary between chunks and enables the model to better understand the sequence as a whole.
In the context of the chunked cross-attention mechanism in the RETRO model, the overlap of chunks ensures that each chunked cross-attention operation can attend not only to the information within its own chunk but also to the neighbouring context from the preceding chunk. This helps to make the attention operation more effective and contextual, as it can utilize a broader range of information when computing attention vectors.

Notes

Longformer: Longformer is a modified version of the Transformer architecture that enables efficient processing of long sequences. It introduces a sliding window attention mechanism that allows each token to attend to a subset of tokens within a fixed range, rather than attending to all tokens. This reduces the quadratic dependency on the sequence length in the standard Transformer and makes it more computationally feasible for long sequences.
Reformer: The Reformer model is designed to address the memory and time cost of Transformers. It introduces the concept of reversible layers, which enable the model to store intermediate activations only for a subset of tokens instead of the entire sequence. This reduces the memory requirements and allows for longer sequences to be processed. Additionally, Reformer employs a technique called locality-sensitive hashing (LSH) for approximate nearest neighbor search, reducing the time complexity of attention computations.
Sparse Transformers: Sparse Transformers focus on reducing the computational cost of the attention mechanism. Instead of attending to all tokens, they only attend to a sparse subset of tokens, selected based on a sparse pattern or using a randomized algorithm. By reducing the number of attended tokens, the computational complexity is significantly reduced, making it more feasible to process long sequences.
Performer: The Performer model proposes an approximation technique for the self-attention mechanism. It replaces the standard quadratic self-attention computation with a linear computation, achieved through the use of the Fast Fourier Transform (FFT). This approximation allows for efficient processing of long sequences while maintaining competitive performance compared to standard Transformers.
Linformer: Linformer is a modification of the Transformer architecture that reduces the computational complexity of the self-attention mechanism. It introduces a linear self-attention pattern that attends to a fixed number of tokens, regardless of the input sequence length. This results in a linear dependency on the sequence length, making it more efficient for long sequences.
These inventions and modifications have contributed to addressing the limitations of Transformers on long sequences and computation memory and time cost, enabling their application in a wider range of tasks and scenarios.

Recall

How gradient checkpoint works? Essentially you trade computation for memory so that you can recompute some intermediately values without saving them to memory.
Use Pytorch to realize GPT

Notes

Gradient checkpointing is a technique used to save memory when training deep learning models. The idea is to not store all intermediate activations of the entire computation graph when you are doing the forward pass of backpropagation, but rather to store a few and recompute the others during the backward pass.
Here's a concrete example:
Let's say you have a very simple deep learning model, a deep fully-connected (dense) neural network with 4 layers, represented as f(g(h(x))). Each function f, g, and h represents a layer in the neural network.
Without gradient checkpointing
In the normal backpropagation process without gradient checkpointing, during the forward pass, you would compute and store the outputs of h(x), g(h(x)), and f(g(h(x))) in memory. Then, during the backward pass, you would use these stored values to compute the gradients.
With gradient checkpointing
If you apply gradient checkpointing, during the forward pass, you would compute the outputs of h(x), g(h(x)), and f(g(h(x))), but you would only store the outputs of h(x) and f(g(h(x))). You don't store the output of g(h(x)).
Then, during the backward pass, you first compute the gradients with respect to the output of f(g(h(x))) as usual. But when you need to compute the gradients with respect to the output of g(h(x)), you don't have it stored in memory. So, you recompute it by doing a forward pass through g(h(x)) again.
This way, you are trading computation (having to recompute g(h(x))) for memory (not storing the output of g(h(x))).
Keep in mind this is a simplified example, and the actual process can be more complex, especially for large models. Also, the best layers to checkpoint can depend on the specific model and training setup.
In PyTorch, you would typically apply gradient checkpointing to a sequence of layers using the torch.utils.checkpoint.checkpoint or torch.utils.checkpoint.checkpoint_sequential functions.

class GPT(nn.Module): def init(self, vocab_size, max_len, num_layers, embed_dim, num_heads, dropout, feedforward_dim): super(GPT, self).init() self.embed = nn.Embedding(vocab_size, embed_dim) self.pos_embed = nn.Parameter(torch.zeros(1, max_len, embed_dim)) self.transformer_blocks = nn.ModuleList([ TransformerBlock(embed_dim, num_heads, dropout, feedforward_dim) for _ in range(num_layers) ]) self.dropout = nn.Dropout(dropout) self.fc = nn.Linear(embed_dim, vocab_size)
